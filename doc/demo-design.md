## 一、核心遊戲規則

### 1.1 一句話定義

> 單人板塊拼放探索遊戲。從城鎮出發，逐張翻開地形板塊、拼出前往崩心的路徑，每塊探明的板塊都帶來一段 AI 生成的敘事。

### 1.2 Beacon Patrol 規則保留與改動

| BP 原版規則 | 保留/改動 | 龍伯版本 | 理由 |
|------------|----------|---------|------|
| 邊緣匹配（水接水、陸接陸） | **保留核心，擴充類型** | 四種邊緣：路/野/墟/裂 | 二元制對敘事承載不足 |
| 固定方向放置（箭頭朝上） | **保留** | 板塊有固定朝向，不可旋轉 | 簡化決策，減少分析癱瘓 |
| 船自動移到新板塊 | **保留** | 龍伯自動移到最後放置的板塊 | 核心節奏感來源 |
| 每回合放 1~3 張 | **改為 1~2 張** | 單人模式節奏較慢，每張牌需要消化敘事 | 配合 AI 敘事的閱讀節拍 |
| 手牌 3 張（1~3 人局） | **保留** | 手牌上限 3 張 | 決策空間剛好 |
| 螺旋槳 token 額外移動 | **改名保留** | 「足印」token，功能相同 | 語境轉換 |
| 棄牌可移動 1 格 | **保留** | 棄掉手中板塊 = 移動 1 格 | 丟棄也有意義 |
| 四面包圍 = 已探索得分 | **改為 3 面 + 事件取代分數** | 3+ 鄰居 = 探明，觸發發現事件 | 單人出牌少，3/4 保持節奏 |
| 多人交換手牌 | **移除** | 單人遊戲不需要 | — |
| 板塊用完就結束 | **改為目標式** | 放出崩口板塊即結束 | Demo 需要明確終點 |
| 計分 → 對照評等 | **改為事典收集** | 探明觸發事典條目，結局展示收集成果 | 分數無法承載敘事 |

### 1.3 四種邊緣類型

| 邊緣 | 字 | 顏色標識 | 敘事意義 | 出典 |
|------|---|---------|---------|------|
| **路** | 路 | 土黃 | 天崩後殘存的交通網絡 | — |
| **野** | 野 | 暗綠 | 城鎮間的蠻荒地帶 | 《列子》「八紘**九野**之水」 |
| **墟** | 墟 | 灰褐 | 崩毀的舊文明建築 | 《列子》「歸**墟**」 |
| **裂** | 裂 | 暗紅發光 | 崩心能量外洩造成的地裂 | — |

**規則**：放置板塊時，所有與已存在板塊相鄰的邊必須類型一致。路接路、野接野、墟接墟、裂接裂。不接觸的邊不受限。

### 1.4 板塊設計

#### 板塊結構

每張板塊 = 一個正方形格子，定義四條邊的類型 + 板塊類別 + 內容。

```
板塊資料結構：
{
  id:         字串        # 唯一識別碼
  category:   枚舉        # settlement / road / wilderness / ruin / approach / entrance
  edges: {
    north:    邊緣類型     # road / wild / ruin / crack
    east:     邊緣類型
    south:    邊緣類型
    west:     邊緣類型
  }
  district:   字串        # 所屬區域名（會稽鎮 / 九野 / 岱輿墟 / 員嶠原 等）
  discovery:  枚舉        # clue / supply / lore / anomaly / none
  narrative_seed: 字串    # 提供給 AI 的情境種子詞
}
```

### 1.6 發現事件系統

當板塊被「探明」（≥3 鄰居）時觸發。

| 發現類型 | 枚舉值 | 效果 | 板塊分佈 |
|---------|--------|------|---------|
| **線索** clue | clue | 事典新增主線相關條目，推進理解 | 聚落、遺址 |
| **補給** supply | supply | 恢復足印 token 1 枚 / 獲得額外抽牌 | 聚落、官道 |
| **掌故** lore | lore | 事典新增世界觀條目（非主線） | 所有類型 |
| **異兆** anomaly | anomaly | 敘事伏筆，暗示地下的危險 | 臨淵、遺址 |
| **無** none | none | 該板塊無特殊發現 | 官道、荒野（部分） |

### 1.7 事典系統（Codex）

**事典 = 遊戲的「分數」等價物。**

```
事典資料結構：
{
  entries: [
    {
      id:        字串
      title:     字串      # 如「崩學概論」「灼骨占卜」
      content:   字串      # AI 生成或預寫的 2~4 句敘事
      category:  枚舉      # mythology / geology / faction / personal
      source_tile: 板塊ID  # 從哪張板塊發現的
      turn:      整數      # 第幾回合發現
    }
  ]
  total_possible: 整數     # Demo 中可收集的最大條目數
}
```

- Demo 中預設 **20 條可收集條目**（不是每張板塊都有）
- 結局畫面顯示「收集 X/20 條」
- 鼓勵重玩（不同放置路徑 → 不同板塊被探明 → 不同條目）

### 1.8 天象牌系統

獨立的小型牌堆，每回合抽 1 張。**不影響棋盤狀態，純敘事功能。**

| 天象類型 | 數量 | 效果 |
|---------|------|------|
| 無事 | 10 | 「風平浪靜。」不顯示任何文字 |
| 環境 | 8 | 氛圍描述：天色、氣溫、遠方聲響 |
| 遭遇 | 5 | NPC 出現在龍伯當前板塊，可互動 |
| 異兆 | 2 | 強烈的伏筆事件：地面震動、天空異象 |

共 25 張，與 25 回合的預估遊戲長度對應。

### 1.9 移動規則

BP 的水路連通改為「共享邊緣連通」：

- 兩張相鄰板塊之間的共享邊必須類型一致（這已由放置規則保證）
- 因此**所有已放置且相鄰的板塊之間都可自由移動**
- 移動消耗：足印 token 或棄牌（每格 1 枚/1 張）
- 足印 token：遊戲開始時 3 枚，補給事件可恢復

### 1.10 結束條件

| 條件 | 類型 | 處理 |
|------|------|------|
| 龍伯移到崩口板塊上 | **抵達**（正常結局） | 播放結局敘事 → 展示事典 → 「待續」 |
| 牌堆空 + 手牌無法放置 | **滯留** | 提供棄牌堆重洗選項（Demo 限定寬容機制） |
| 玩家主動選擇撤退 | **撤退** | 展示已收集事典 → 「也許下次會走得更遠」 |

**Demo 無失敗狀態。** 目的是 hook 玩家，不是懲罰。


### 2.3 關鍵演算法

#### 邊緣匹配檢查

```
can_place(tile_def, position, board):
    for each direction in [north, east, south, west]:
        neighbor_pos = position + direction_vector
        if neighbor_pos in board:
            my_edge = tile_def.edges[direction]
            neighbor_edge = board[neighbor_pos].edges[opposite(direction)]
            if my_edge != neighbor_edge:
                return false
    // 至少要有一個鄰居（不能放到孤島）
    if no neighbors found:
        return false
    // 必須與龍伯相鄰（BP 的「船所在板塊的正交方向」規則）
    if position not adjacent to player.position:
        return false
    return true
```

#### 探明檢查

```
check_exploration(board):
    newly_explored = []
    for each (position, placement) in board:
        if placement.explored:
            continue
        neighbor_count = count of occupied orthogonal neighbors
        if neighbor_count >= 3:
            placement.explored = true
            placement.explored_turn = current_turn
            newly_explored.append(placement)
    return newly_explored
```

#### 三段式牌堆初始化

```
init_deck(all_tiles):
    segment_1 = filter(all_tiles, category in [settlement, road] AND no crack edges)
    segment_2 = filter(all_tiles, category in [road, wilderness, ruin] AND ≤1 crack edge)
    segment_3 = filter(all_tiles, category in [ruin, approach, wilderness] AND has crack edges)

    shuffle(segment_1)
    shuffle(segment_2)
    shuffle(segment_3)

    // 段 1 在頂，段 3 在底
    deck = segment_1 + segment_2 + segment_3
    return deck
```

```
    └────┬────┘                            │
         │                                 │
```

### 2.5 AI 敘事介面規格

**設計原則**：「AI 詮釋不決定」— AI 不影響任何遊戲數值，只產生文字。

#### 呼叫時機與 I/O

| 呼叫點 | 輸入 | 輸出 | 頻率 |
|--------|------|------|------|
| **板塊探明** | tile_def + 周邊板塊 + codex 摘要 + turn | narrative (str) + codex_entry? + mood (enum) | 每 2~4 回合 |
| **NPC 遭遇** | npc_id + npc_faction + codex 摘要 | dialogue (str) + choices (list) + codex_entry? | 每局 3~5 次 |
| **天象描述** | turn + player_distance_to_entrance + board_size | atmosphere_text (str) | ~30% 的回合 |
| **結局敘事** | 完整 codex + total_turns + path_summary | ending_narrative (str) | 每局 1 次 |

#### tool_use schema（Claude API）

```json
{
  "name": "discovery_event",
  "input_schema": {
    "properties": {
      "narrative":     { "type": "string", "description": "2~3 句發現敘事" },
      "discovery_type": { "enum": ["clue", "supply", "lore", "anomaly"] },
      "codex_entry": {
        "type": "object",
        "nullable": true,
        "properties": {
          "title":    { "type": "string" },
          "content":  { "type": "string" },
          "category": { "enum": ["mythology", "geology", "faction", "personal"] }
        }
      },
      "mood": { "enum": ["neutral", "ominous", "hopeful", "mysterious", "unsettling"] }
    }
  }
}
```

**離線備案**：每種板塊類別 × 每種發現類型 預寫 1 條靜態文字，共約 30 條。API 失敗時無縫替代。

## 四、Demo 遊玩流程

### 開場
- 標題畫面 → 序言：「天崩紀元第十年。你叫龍伯，是湯問學宮崩學研究所的博士生……」
- 起始板塊（會稽鎮）已在場上，手牌 3 張，崩口板塊面朝上展示在旁

### 第一幕：城鎮（回合 1~8）
- 牌堆頂段：聚落 + 官道
- 探索湯問學宮 → 事典條目：崩學概論
- NPC：導師，警告崩心
- 教學：低風險學會邊緣匹配

### 第二幕：行路（回合 9~18）
- 牌堆中段：荒野 + 遺址
- 天象事件漸強：轟鳴、地震
- 遺址（岱輿墟）→ 事典：天崩前文明
- 裂邊開始出現 → 視覺暗示接近

### 第三幕：臨淵（回合 19~25）
- 牌堆底段：臨淵 + 裂邊密集
- 色彩去飽和，地裂紋理
- NPC：禺強信使勸退
- 拼出裂邊路徑 → 放置崩口

### 結局
- 崩口板塊放置 → 龍伯移動至其上
- AI 生成結局敘事（基於收集的事典）
- 展示事典收集成果 X/20
- 「待續……」

**預估時長：25~35 分鐘**

## 五、引擎選型 — Godot 4

已選定 **Godot 4.6（GDScript）**。決定理由：

- **全專案文字檔**（.tscn / .gd / .tres），AI 可直接讀寫，協作效率最高
- GDScript 語法接近 Python，學習曲線低
- 完全免費、MIT 授權
- GodotSteam 可處理 Steam 整合
- 社群快速成長，插件生態完善
